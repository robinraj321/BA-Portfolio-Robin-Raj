## 🛒 Project 1 — E-Commerce Checkout Optimization

### Problem Statement
ShopEase, a mid-sized Indian e-commerce platform with **50,000+ monthly visitors**, was losing **65% of users** at the checkout stage — resulting in an estimated **Rs. 2,47,000 monthly revenue loss**.

### My Role
Business Analyst — responsible for full analysis from problem identification to solution documentation.

### What I Did

#### 🔍 Discovery & Analysis
- Analyzed website funnel data using Google Analytics and Hotjar session recordings
- Identified a **65% cart abandonment rate** with the steepest drop (44%) between "Add to Cart" and "Checkout Started"
- Conducted **6 elicitation sessions**: stakeholder interviews, user surveys (n=150), usability testing, competitor benchmarking (Amazon, Flipkart, Myntra), web analytics review, and document analysis

#### 🧩 Root Cause Analysis
Applied Fishbone (Ishikawa) framework and identified **3 critical friction points**:
1. **Mandatory account registration** — 34% of abandonments
2. **Lengthy form fields (12 fields)** — 28% of abandonments
3. **Payment processing delays (8–12 sec)** — 20% of abandonments

#### 📋 Requirements & Documentation
- Mapped **7 stakeholders** with influence/interest matrix and RACI
- Documented **15 Functional Requirements** using MoSCoW prioritization across 5 modules: Auth, Form, Payment, Trust, Flow
- Created full **Business Requirements Document (BRD)** with process flow diagrams

#### 🏃 Agile Delivery
- Wrote **12 Agile User Stories** with acceptance criteria and story points (Planning Poker)
- Planned **4 Sprints** (8 weeks): Auth & Address → Payment & Validation → Flow & Mobile → Biometric & UAT
- Achieved **94.3% sprint velocity** (33/35 story points delivered)

#### 📈 Proposed Solution
Simplified **2-step checkout flow** (Details → Payment) eliminating all 3 friction points.

### Results / Projected Impact

| Metric | Before | After (Projected) | Change |
|--------|--------|-------------------|--------|
| Cart Abandonment Rate | 65% | 46% | -29% |
| Checkout Steps | 6 | 2 | -66% |
| Form Fields | 12 | 5 | -58% |
| Avg. Checkout Time | 180 sec | 90 sec | -50% |
| Monthly Revenue | Rs. 3,80,000 | Rs. 4,46,500 | +Rs. 66,500 |

### Skills Demonstrated
`Requirement Elicitation` `Stakeholder Analysis` `Funnel Analysis` `Root Cause Analysis` `BRD Documentation` `User Stories` `MoSCoW Prioritization` `Agile/Scrum` `Process Flow Diagrams` `Risk Register`

## 👥 Project 2 — HR Onboarding Process Improvement

### Problem Statement
TechServe India Pvt. Ltd. (850+ employees) had new hires taking **8–11 days on average** to become fully operational due to redundant approval steps and manual processes — costing **Rs. 45,000/year** in lost productivity. **23% of early attrition** cases cited poor onboarding as a contributing factor.

### My Role
Business Analyst — responsible for As-Is process mapping, gap analysis, requirements gathering, and To-Be process design.

### What I Did

#### 🔍 Discovery & Analysis
- Mapped the **As-Is onboarding process** using BPMN notation across 8 sequential steps and 3 departments (HR, IT, Department Management)
- Conducted **6 elicitation techniques**: structured interviews, process observation (2-day shadow), new hire surveys (n=47), document review, SHRM benchmarking, and a full-day JAD workshop
- Measured actual onboarding durations per department: Engineering (10.2d), Finance (11.3d), Operations (9.1d)

#### 🧩 Root Cause Analysis
Identified **5 root causes** using Fishbone framework:

| Root Cause | Category | Days Lost |
|------------|----------|-----------|
| 4 sequential approvals (2 redundant) | Process | 3.2 days |
| 100% manual document handling | Technology | 1.8 days |
| IT setup waits for all approvals | Process/Tech | 1.5 days |
| Manager unavailability | People | 0.9 days |
| No centralized tracking | Communication | 0.6 days |

#### 📋 Requirements & Documentation
- Mapped **9 stakeholders** using RACI framework across 4 organizational levels
- Documented **14 Functional Requirements** across modules: Portal, Documents, Approvals, IT Provisioning, Tracking, Induction, Reporting, Integration
- Delivered full **As-Is vs To-Be Gap Analysis** across 8 process dimensions

#### 🏃 Agile Delivery
- Wrote **12 Agile User Stories** covering 5 personas: New Hire, HR Manager, IT Manager, Department Manager, HR Head
- Planned **4 Sprints** (8 weeks): Portal & Docs → Approvals & IT → Dashboard & Induction → HRIS & Mobile
- Achieved **95% sprint velocity** (38/40 story points delivered)

#### ⚙️ Proposed Solution
Key architectural changes in the To-Be design:
- **Eliminated 2 redundant approval stages** (4 → 2 approvals)
- **IT provisioning runs in parallel** with HR approval (not after)
- **100% digital document portal** with auto-validation
- **Real-time onboarding status tracker** for new hires

### Results / Projected Impact

| Metric | Before | After (Projected) | Change |
|--------|--------|-------------------|--------|
| Avg. Onboarding Duration | 8–11 days | 3.5 days | -57% |
| New Hire Satisfaction | 5.8/10 | 8.6/10 | +48% |
| Manual HR Hours per Hire | 6.5 hrs | 1.5 hrs | -77% |
| Document Error Rate | 34% | <5% | -85% |
| Annual Process Cost | Rs. 45,000 | Rs. 18,000 | Saves Rs. 27,000/yr |
| Early Attrition (Onboarding) | 23% | <10% | -57% |
